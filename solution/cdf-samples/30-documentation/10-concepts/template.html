Concepts

This framework provides a standardized way to implement and publish dashboards to the Pentaho BI Platform.  The main goal of this framework is to give the user an easy and friendly way to build dashboards..
How can this be achieved? Take a look on this advantages:

  • No need to create additional JSP pages to deliver new dashboards.

    • The user has no need to make jsp pages or even have the knowledge to deal
    with this kind of technology.

  • No need to create pentaho Action Sequences.

    • For most cases,the user only has to set the needed components using a friendly syntax.

  • The dashboard objects can be easy reused.

    • The behaviors of dashboard objects (components) are related to their parameters values, and so
    the user may use the same object on an different dashboard, modifying only the values
    of one or two parameters.

  • Auto refresh and interaction between dashboard objects.

    • The user doesn't need to concern about interaction between components. The only thing
    the user must specify is what variable is bound to the object.

  • The ability to auto generate a navigation bar that picks up new dashboards.

    • A new dashboard was added? No need do modify the menu, the navigation bar will
    deal and present the new one.

  • New form objects, timeplot charts, tooltips, openstreet maps integration, etc, that makes
    the dashboard richer.

    • What is need to get all this new stuff to work? Just add the desired component.

  • Build your one dashboard component library.

    • When a particular component with a specific behavior is being used in all your projects,
    just add it to your javascript library for later use.

Beyond this, the are other technical advantages that this framework brings to pentaho dashboard:

  • Enforces security via Pentaho's security model.
  • Separation of the html design and component definition.
  • Portlet integration.
