Template Files

Once again, the main goal is to reduced time and effort to make a new dashboard. The dashboard template files were build to avoid adding always the same content (Navigation bar, javascript libraries, header, etc) to all the dashboards templates files. Another objective that was accomplished with this solution was to avoid the creation of a template file for all the solution folders. Finally, but not less important, all the libraries present in the template are dynamically added by the Dashbaord.jsp reducing the size and computation of the templates.

Here are the tree types of template files, used by cdf framework, that are concatenated to the user template files:

  • template-dashboard: This template contains the header, the navigation bar and all the necessary JavaScript libraries

  • template-dashboard-mantle: This template contains the header and all the necessary JavaScript libraries. The navigation menu was excluded, because mantle already have is own navigation approach.

  • default-dashboard-template: When no template is present in the current solution folder, this template is present to the user. Can be used to present a content navigation page

Two questions are raised here:

How can we choose beteween the template-dashboard-mantle and the template-dashboard? The answer is simple, just invoke the dashboard.jsp with the template request argument.(example: template=mantle).

Can i modify the templates? Of course, that's the purpose of template files. Just don't forget to include the navigatorComponent.
