Components

The Component concept was born to simply the building of dashboard objects. Therefor
a Component is just a simple JavaScript object that encapsulate all the object properties
and behaviors. But how much effort is necessary to make a component that:

  • Have a different behavior before and after execution.
  • That interacts with other dashboard components
  • At start up is just a simple bar chart but can be changed dynamically to a dial chart or even a table.
  • The result presented by the component should change dynamically according to one or more parameter values.


The next paragraphs and chapters will show that all of this can be accomplished with very little effort.

Basic Structure

All components have a set of "properties" that are common to almost all of them and that are used to
control their life cycle and behavior. Before the enumeration of this "properties", let's clarify
what is the life cycle of the component:

Life cycle diagram

Component Properties

  • This property assumes a variety of values like jfreechart, timeplot, datepicker, etc.

  • Name: This is the identifier of the component.

  • Parameters: The parameter(s) is an array of identifiers that is used by the component to get or store the desirably value in the correspondent object.

  • Listeners: The variables that triggers the component update are stored in this array. This array is crucial in the iteration between components, because they can communicate using this together with their parameters. For example, if the selected value(parameter) of one selector changes, all the components that are "listening" on this parameter will react..

  • PreExectuion/PreChange: Before the component is updated or presented to the user, this actions(functions) are executed. (The PreChange applies to form components and PreExections to the other ones).

  • PostExectuion/PostChange: After the component is updated or presented to the user, this actions(functions) are executed. (The PostChange applies to form components and PostExectuion to the other ones).

  • Chart Definition: All the chart definitions(parameters) are defined in this javascript object.

  • HTML Object: This is the identifier of the html object that will be replaced by the component content result.

  • ExecuteAtStart: If set to false, the component will not execute at start. Although it will be updated as soon as one of his listeners have changed
